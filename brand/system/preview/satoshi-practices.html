OpenType — features que importam

Satoshi expõe `ss01` (alt 'a' single-story), `tnum` (números tabulares), `cv01` (zero cortado). Liga sempre o que faz sentido para o contexto.

0→0

Weight pairing — como combinar

Pares calibrados. Tudo o que sair daqui parece descoordenado. Light (300) é proibido em pares — só callout isolado.

Não fica bom Light + Black ✕ Contraste de peso muito violento em Satoshi — quebra a hierarquia.

Regras — do / don't

  • Use o variable font em produção. 1 request vs 10 arquivos.
  • Aplique tnum em KPIs, tabelas e qualquer coluna de números — colunas alinham.
  • Tracking aperta com tamanho: -0.05em em 100px+, 0 em body. Sempre.
  • Italic = Satoshi italic real, não font-style: italic no Regular. O TTF italic já está carregado.
  • Não use peso 400 para headings — fica fraco em Satoshi. Mínimo: 500.
  • Não use Light (300) em corpo de texto — fica frágil em telas pequenas e dificulta leitura.
  • Não use tracking positivo em body — Satoshi é desenhada para tracking 0 ou ligeiramente negativo.
  • Não italize UI (labels, buttons, chips, tabs) — italic é para texto editorial.
