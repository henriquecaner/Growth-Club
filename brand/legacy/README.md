# brand/legacy/ — artifacts arquivados

Estes arquivos foram o **Chunk 5** do brand brief plan original (consolidação textual do brand book v1, gerado pré-design-system). Foram arquivados em **2026-05-17** quando o `brand/system/` (Growth Club Design System) se tornou a fonte canônica de implementação.

## Arquivos

- `brand-book-v1.html` — versão HTML do brand book consolidado (visualização web)
- `brand-book-v1.pdf` — versão PDF (545KB, com tipografia + paleta aplicadas)
- `brand-book.md` — fonte Markdown que gerou os outros dois
- `brand-book.css` — CSS específico do brand book HTML antigo

## Por que estão arquivados

1. **Conteúdo conceitual já replicado** em `brand/system/README.md` (que é a narrativa viva do design system).
2. **Decisões visuais (paleta/tipografia)** foram reescaladas via AD-008 (2026-05-17 em `.specs/project/STATE.md`):
   - Archivo Black → Satoshi
   - Amber Beer rebalanceado de 3% acento → ~10% CTA primário
3. **Riscar de leitura desatualizada** — manter o PDF acessível no topo de `brand/` gerava tentação de tratar como verdade quando o sistema já evoluiu.

## Quando reabrir

- Histórico estratégico: como a marca foi inicialmente articulada antes do design system entrar.
- Comparação com versões futuras: se um dia for gerar `brand-book-v2`, este é o ponto de partida documentado.
- Audit de promessas: garantir que `brand/system/` não inventou nada que não estivesse nas decisões originais.

**Não usar como fonte de design** — para tokens, cores, tipografia, voz canônica, ir em `brand/system/` + `brand/visual/*` + `brand/voice/*`.
